# DroidDisassembler

**Кроссплатформенный инструмент для реверс-инжиниринга (в разработке)** — дизассемблирование и декомпиляция нативных и управляемых библиотек в читаемый псевдокод.

[![Python](https://img.shields.io/badge/Python-3.8+-blue.svg)](https://python.org)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

## ✨ Возможности

- Поддержка форматов **ELF** (ARM/ARM64/x86/x64) и **PE** (x86/x64)
- Работа с управляемым кодом: **IL2CPP** (глобальные метаданные) и Mono/.NET
- Автоматический анализ потока управления (CFG) и генерация структурированного псевдокода
- Декодирование C++ имён (demangling) через `c++filt`
- Навигация по функциям, экспортам и секциям
- Поиск, переименование и экспорт функций
- Настраиваемые темы (светлая/тёмная) и шрифты
- 
## 📋 Требования

- Python 3.8+
- Библиотеки из `requirements.txt`

```bash
pip install -r requirements.txt
```

| Библиотека | Назначение |
|-----------|------------|
| `capstone` | Дизассемблер |
| `pyelftools` | Парсинг ELF |
| `pefile` | Парсинг PE |
| `PySide6` | Графический интерфейс |

## 🚀 Запуск

```bash
python droid_disassembler.py
```

## 📖 Использование

1. **Файл → Открыть** — выберите `.so`, `.dll` или исполняемый файл
2. Выберите функцию в дереве слева для декомпиляции
3. Для IL2CPP — положите `global-metadata.dat` рядом с библиотекой
4. **Alt+T** — быстрый поиск по функциям

## ⚠️ Статус проекта

Проект находится **в активной разработке**. Некоторые функции могут работать нестабильно:

- Декомпиляция ARM/ARM64 частично реализована
- Псевдокод для сложных функций требует доработки
- Поддержка Mono ограничена
